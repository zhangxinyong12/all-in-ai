### 阶段2：Prompt Engineering（1.5个月）

#### 学习目标
- 掌握提示词设计技巧
- 理解提示词工程最佳实践
- 能够设计复杂提示词模板

#### 核心内容

**1. 提示词基础结构**
```
角色设定 + 任务描述 + 输入数据 + 输出格式 + 约束条件 + 示例
```

**2. 常用技巧**
- Zero-shot：零样本提示
- Few-shot：少样本提示（提供示例）
- Chain-of-Thought：思维链（让模型逐步思考）
- Self-Consistency：自我一致性验证

**3. 实战技巧**
- 角色扮演设定
- 分步分解复杂任务
- 使用XML/JSON结构化输出
- 添加验证和纠错逻辑

**示例模板**
```
你是一位资深的前端开发专家。

任务：审查以下React代码，指出潜在问题和改进建议。

代码：
{code}

输出要求：
1. 以JSON格式输出
2. 包含以下字段：
   - issues: 问题列表
   - suggestions: 改进建议
   - rating: 代码评分(1-10)

输出示例：
{
  "issues": [...],
  "suggestions": [...],
  "rating": 7
}
```

**学习资源**
- OpenAI Prompt Engineering Guide
- Anthropic Prompt Library
- Learn Prompting 网站
- 《提示工程指南》中文版

**实践任务**
- 设计代码审查提示词
- 设计文档生成提示词
- 设计需求分析提示词
- 构建一个提示词管理系统

### 阶段3：RAG技术（1.5个月）

#### 学习目标
- 理解RAG原理和应用场景
- 掌握向量数据库使用
- 能够构建完整的RAG系统

#### 核心内容

**1. RAG基本原理**
```
用户问题 → 向量化 → 向量检索 → 相关文档提取 → 与问题组合 → LLM生成答案
```

**2. 文本向量化**
- Embedding模型选择
- 文本切片策略
- 向量相似度计算

**3. 向量数据库**
- ChromaDB（轻量，适合入门）
- Pinecone（云服务）
- Milvus（开源企业级）
- Weaviate

**4. RAG优化技巧**
- 重排序（Reranking）
- 混合检索（关键词+向量）
- 文档元数据过滤
- 上下文窗口管理

**技术栈选择**
```
Python + LangChain + ChromaDB + Streamlit
或
Node.js + LangChain.js + Pinecone + React
```

**示例代码**
```python
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import Chroma
from langchain.chains import RetrievalQA

# 创建向量存储
vectorstore = Chroma.from_documents(
    documents=docs,
    embedding=OpenAIEmbeddings()
)

# 创建RAG链
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    retriever=vectorstore.as_retriever()
)
```

**学习资源**
- LangChain官方教程
- Pinecone RAG教程
- 《Retrieval-Augmented Generation》论文

**实践任务**

- 实现代码库智能搜索
- 开发产品文档助手
- 优化检索准确率（目标>85%）

### 阶段4：AI Agent开发（1个月）

#### 学习目标
- 理解Agent设计模式
- 掌握工具调用和函数调用
- 能够构建多Agent协作系统

#### 核心内容

**1. Agent基础概念**
- 核心组件：感知、规划、行动、记忆
- 工具使用（Function Calling）
- 自主决策机制

**2. 主流框架**
- LangChain Agents
- AutoGPT
- BabyAGI
- CrewAI（多Agent协作）

**3. 常用工具集成**
- 文件操作：读写、搜索
- 网络请求：API调用
- 数据库：SQL查询
- 代码执行：Python REPL

**4. Agent设计模式**
- ReAct（推理+行动）
- Plan-and-Solve
- Self-Refine
- Multi-Agent协作

**示例架构**
```python
from langchain.agents import initialize_agent, Tool
from langchain.llms import OpenAI

# 定义工具
tools = [
    Tool(
        name="Search",
        func=search_func,
        description="搜索互联网信息"
    ),
    Tool(
        name="Calculator",
        func=calculator_func,
        description="执行数学计算"
    )
]

# 创建Agent
agent = initialize_agent(
    tools,
    OpenAI(),
    agent="zero-shot-react-description"
)
```

## 七、常见问题与建议

### Q1: 需要学Python吗？

**A:** 推荐学习。虽然可以用Node.js，但Python是AI生态的主流语言，学习成本不高（1-2周入门）。

### Q2: 需要深入数学吗？

**A:** 应用开发不需要高深数学。了解基础概念即可：向量、矩阵、概率统计。

### Q3: 大专学历会被卡吗？

**A:** 应用开发岗位更看重实战能力。作品集和项目经验比学历更重要。建议：
- 完成高质量项目
- 参与开源贡献
- 建立技术影响力（博客、演讲）

### Q4: 如何快速验证学习效果？

**A:** 每个阶段后输出：
- 技术博客文章
- 可演示的Demo
- GitHub开源项目

### Q5: 学习时间不够怎么办？

**A:** 最小可行方案：
- 专注Prompt工程和API集成（3个月）
- 完成1个高质量项目
- 优先投递对学历要求宽松的公司
